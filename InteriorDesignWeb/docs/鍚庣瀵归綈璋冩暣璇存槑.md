本次不是重做原有功能，而是对当前后端做一次结构收口：

1. 保留已有可运行的 `/api/flux/*` ComfyUI 生图链路。

## 当前 AI 生图主链路

当前真实生图仍然是：

```text
AIGenerationController /api/flux/generate
  ↓
ComfyUIService.SubmitGenerationJob()
  ↓
FluxWorkflowService.GenerateWorkflow()
  ↓
ComfyUI API
  ↓
轮询 /history
  ↓
CosService 上传图片
  ↓
aigenerationjobs / aigenerationjobimages
```

新的 `AIJobsController` 当前只是任务中心骨架，暂不调用 ComfyUI。

---

## 后续推荐合并方向

未来应逐步变成：

```text
AIJobsController
  ↓
AIJobOrchestrator
  ↓
WorkflowRegistry / WorkflowBuilder
  ↓
ComfyUIProvider
  ↓
AIResultService
  ↓
aigenerationjobs / aigenerationjobimages / projectimages
```

其中：

| 模块 | 未来定位 |
|---|---|
| `AIJobsController` | 新统一 AI 生图入口 |
| `ComfyUIService` | 逐步拆分，最终降级为 Provider |
| `FluxWorkflowService` | 逐步升级为配置化 WorkflowBuilder |
| `JobTrackingService` | 合并进 AIJobService 或保留为兼容层 |
| `ProjectImagesController` | 继续负责“图片加入项目” |

## 7 个工作流接入前必须准备

接入 7 个工作流前，需要提供：

1. 7 个工作流名称和用途。
2. 每个工作流 JSON。
3. 每个工作流上传 API。
4. 每个工作流输入参数。
5. 每个工作流输出格式。
6. 每个工作流节点 ID / input key 映射。
7. 哪些工作流需要原图、参考图、mask。
8. 每个工作流预计使用模型和成本策略。
